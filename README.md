# Stego Project: Attack, Detection, and Message Extraction

A local steganography lab that supports:
- Attack (embed a message into an image)
- Detection (binary clean/stego + multiclass technique prediction)
- Message extraction (with technique-aware decoding and fallback)
- Web UI with Streamlit for interactive testing

This project uses two pretrained PyTorch checkpoints:
- `binary_model.pth` for clean vs stego classification
- `multiclass_model.pth` for technique classification

These PyTorch checkpoints are used with the `cyber-fixed` workflow/notebook.

Optional additional model artifact in this repo:
- `model_final_7classes.keras` (and `model_final_7classes.h5`) can be tested separately if you want to experiment with a different 7-class model pipeline.
  These Keras models are intended for the `cyber-fixed-2` notebook workflow.
  Note: the current `demo.py` and `streamlit_app.py` are wired to the PyTorch `.pth` checkpoints by default.

---

## Features

- Supports 5 embedding techniques in the local demo pipeline:
  - LSB
  - SSB-4
  - SSB-N
  - DCT
  - FFT
- CLI workflow in `demo.py`:
  - `attack`
  - `detect`
  - `evaluate`
- Streamlit dashboard in `streamlit_app.py`:
  - Attack tab
  - Detect tab
  - Evaluate tab
- Automatic extraction fallback:
  - tries hinted/predicted technique first
  - then tries all supported techniques if needed

## Environment Setup

### 1. Create and activate virtual environment

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 2. Install dependencies

If you do not already have them:

```powershell
pip install torch torchvision pillow numpy requests streamlit
```

## CLI Usage

## Detect

```powershell
python demo.py detect --image "cat_stego.png"
```

With technique hint:

```powershell
python demo.py detect --image "cat_fft.png" --technique fft
```

## Attack

```powershell
python demo.py attack --image "cat.jpg" --message "this is our cyber project" --technique lsb --output "cat_lsb.png"
```

DCT example:

```powershell
python demo.py attack --image "cat.jpg" --message "secret dct message" --technique dct --output "dct.bmp"
```

FFT example:

```powershell
python demo.py attack --image "cat.jpg" --message "secret fft message" --technique fft --output "cat_fft.png"
```

## Evaluate (single image)

```powershell
python demo.py evaluate --image "cat_lsb.png"
```

## Evaluate (folder)

```powershell
python demo.py evaluate --dir "results"
```

## Streamlit UI

Run the app:

```powershell
.\.venv\Scripts\python.exe -m streamlit run streamlit_app.py
```

Then open the local URL shown in terminal (usually `http://localhost:8501`).

### UI Sections

- Attack:
  - upload/local sample
  - choose technique and message
  - generate and download stego image
- Detect:
  - run binary + multiclass prediction
  - extract message
  - display fallback-decoding source
- Evaluate:
  - batch analysis of multiple images
  - summary counts + per-file table

## Important Notes

- The classifier output is based on pretrained models and can misclassify custom-generated samples.
- Extraction is independent from classification confidence and includes fallback attempts.
- If your filename contains spaces or parentheses, always quote it in PowerShell:

```powershell
python demo.py detect --image "catsteg (1).jpg"
```

## Troubleshooting

## FileNotFoundError for image path

Use quoted path and verify file exists in current directory.

## Message returns `(none)`

- Try explicit hint:

```powershell
python demo.py detect --image "your_image.png" --technique dct
```

- Ensure the image was generated by this project pipeline.
- Lossy formats may damage payload. Prefer PNG/BMP for testing.

## Model files missing

Make sure both are in project root:
- `binary_model.pth`
- `multiclass_model.pth`
